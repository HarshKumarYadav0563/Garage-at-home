You are a Next.js (App Router) + React + TailwindCSS + Framer Motion + Zustand engineer.
Improve the CART experience on the Services pages so that when a service is selected:
1) a floating Cart icon appears with item count + est. total,
2) clicking it opens a summary panel,
3) user can proceed to booking.

--------------------------------
Goals
--------------------------------
- Floating cart icon with badge (count) that appears only when ≥1 item selected.
- Click/press on the icon opens a **Cart Summary Drawer** (mobile: bottom sheet; desktop: right sidebar).
- Summary lists selected services + add-ons with city-adjusted price ranges, quantity (if applicable), and remove action.
- CTA buttons: **Continue → Details** (go to next booking step) and **Clear all**.
- Persist cart state in localStorage and sync with existing selection logic.

--------------------------------
Tech & State
--------------------------------
- State: Zustand store `useCartStore` (coexists/merges with existing `useBookingStore`):
  - services: `{ id, title, priceMin, priceMax, vehicle, city }[]`
  - addons:   `{ id, title, priceMin, priceMax }[]`
  - totals:   `{ min: number, max: number }` (computed selector)
  - actions:  `addService`, `removeService`, `toggleService`, `addAddon`, `removeAddon`, `clear`, `hydrate()`
  - UI:       `isCartOpen`, `openCart()`, `closeCart()`
- Persistence: hydrate from localStorage on mount (SSR-safe guard).
- Pricing helper: reuse/apply city multipliers before computing totals.

--------------------------------
Floating Cart Icon (FAB)
--------------------------------
- Position: `fixed bottom-4 right-4 md:bottom-6 md:right-6 z-50`
- Style: dark glass pill with gradient ring; Tailwind example:
  `rounded-full bg-white/5 backdrop-blur-xl border border-white/10 shadow-lg h-14 w-14 grid place-items-center`
- Badge: count of selected services (not addons) in a small pill at top-right.
- Animation:
  - On first add: bounce-in (`scale` spring) + confetti-like sparkle (subtle).
  - While items change: tiny pulse.
  - Keyboard shortcut: `⌘/Ctrl + B` toggles cart.

--------------------------------
Cart Summary Drawer/Sidebar
--------------------------------
- Mobile: **Bottom sheet** (full-width): `fixed inset-x-0 bottom-0 rounded-t-2xl bg-black/90 backdrop-blur-xl border-t border-white/10`
- Desktop: **Right sidebar**: `fixed right-0 top-0 h-dvh w-[380px] bg-black/80 backdrop-blur-xl border-l border-white/10`
- Framer Motion:
  - Mobile: slide up from `y:100% → 0`
  - Desktop: slide in from `x:100% → 0`
  - Dim overlay with fade (`fixed inset-0 bg-black/40`)
- Accessibility:
  - `role="dialog"` with `aria-labelledby="cart-title"`
  - Trap focus inside while open; ESC closes.
  - Close button with `aria-label="Close cart"`

--------------------------------
Cart Content
--------------------------------
Header:
- Title: “Your Selection”
- City chip (e.g., “Delhi — Delhi-NCR”) and vehicle chip (Bike/Car).
- Small note: “Prices shown are **estimates** (parts extra).”

Line Items:
- For each service: title, optional subtitle, **₹min–₹max** (after city factor).
- Remove (×) button per item.
- Add-ons appear below lines with a separate heading; allow toggle on/off.

Totals:
- Est. subtotal **₹X–₹Y** (sum of ranges).
- If city factor > 0: “Includes city factor +5% for Gurugram”.

CTAs:
- Primary: **Continue → Details** (navigates to the current booking flow step; keep URL `/services/[vehicle]/[city]#details` or router push to step 2).
- Secondary: **Clear all** (empties store, hides cart icon).
- Tertiary: “Add more services” (closes cart).

Empty state:
- Illustration + “No services selected yet.”
- Button: “Browse services” (scroll to grid, close drawer).

--------------------------------
Interactions & Triggers
--------------------------------
- When user taps **Add** on a service card:
  - Add to cart, show FAB (if hidden), play bounce animation.
- Clicking FAB opens drawer; clicking overlay or close button closes it.
- Pressing ESC closes when open.
- On route change to the next step, keep cart summary as a collapsible panel in sidebar (desktop) or step header (mobile).

- Buttons: primary gradient with glow hover; secondary ghost/glass.
